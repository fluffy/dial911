Abstract

This specification defines how to use Session Description Protocol (SDP) to signal that media will be transported over Datagram Transport Layer Security (DTLS) and it defines new SDP protocol identifiers. This allows the security provided by the existing TCP/TLS specifications for stream transported media to also be applicable to media that is transported over a datagram oriented transport.

1. Introduction

Session Description Protocol (SDP)[2] (Handley, M. and V. Jacobson, “SDP: Session Description Protocol,” April 1998.) has been used to set up the transport of various types of media with RTP over UDP[7] (Schulzrinne, H. and S. Casner, “RTP Profile for Audio and Video Conferences with Minimal Control,” July 2003.), TCP[4] (Lazzaro, J., “Framing RTP and RTCP Packets over Connection-Oriented Transport,” September 2005.), and TLS[3] (Lennox, J., “Connection-Oriented Media Transport over the Transport Layer Security (TLS) Protocol in the Session Description Protocol (SDP),” September 2005.). DTLS[5] (Rescorla, E. and N. Modadugu, “Datagram Transport Layer Security,” June 2005.) is a protocol for applying TLS security to datagram protocols such as UDP and DCCP[6] (Kohler, E., “Datagram Congestion Control Protocol (DCCP),” March 2005.). This specification defines new SDP protocol identifiers that allow SDP to indicate that DTLS should be used to transport the media.

The handling of TLS session in SDP is defined in [3] (Lennox, J., “Connection-Oriented Media Transport over the Transport Layer Security (TLS) Protocol in the Session Description Protocol (SDP),” September 2005.) which discusses only TLS over TCP. This document only extents that specification to also deal with TLS over datagram protocols such as UDP and DCCP.

3. Session Description for RTP/AVP over DTLS

This specification defines new tokens to describe the protocol used in SDP "m=" lines. The new values defined for the proto field are:

  • When a RTP/AVP stream is transported over DTLS with DCCP, then the token SHALL be DCCP/TLS/RTP/AVP.
  • When a RTP/AVP stream is transported over DTLS with UDP, the token SHALL be UDP/TLS/RTP/AVP.
  • When a RTP/AVP stream is transported over TLS with TCP, the token SHALL be TCP/TLS/RTP/AVP.
  • When media is transported over DTLS with UDP, the token SHALL be UDP/TLS.
  • When media is transported over DTLS with DCCP, the token SHALL be DCCP/TLS.

For RTP profiles other than AVP, a new token should be defined in the form of DCCP/DTLS/RTP/xyz where xyz is replaced with an appropriate token for that profile.

Jacobson, “SDP: Session Description Protocol,” April 1998.) [2]. Specifically it adds the following values to the table for the "proto" field.

                DCCP/TLS/RTP/AVP             [RFC-XXXX]
                UDP/TLS                      [RFC-XXXX]
                DCCP/TLS                     [RFC-XXXX]
